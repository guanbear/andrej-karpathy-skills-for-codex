# Contributing

Thanks for helping improve this project.

## Principles

- Keep the skill short enough to be useful in context.
- Prefer concrete coding-agent behavior over motivational phrasing.
- Keep attribution visible and accurate.
- Do not copy upstream wording wholesale.
- Keep the project Codex-native unless a broader adapter strategy is explicitly discussed.

## Local Checks

Run the validation script before opening a pull request:

```bash
node scripts/validate.mjs
```

## Pull Requests

Good pull requests usually do one of these:

- tighten the skill instructions,
- improve install or usage docs,
- fix Codex plugin metadata,
- improve attribution clarity,
- add a focused validation check.

Avoid unrelated formatting sweeps or broad rewrites.
